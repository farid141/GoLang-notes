goroutine adalah sebuah thread ringan, 2:1000. Bisa menjalankan 100000 go routine secara bersamaan. Jika di Java tidak akan bisa sampai angka tersebut


go routine berjalan secara concurrent (beberapa task bergantian dalam satu thread) yang diatur oleh go scheduler untuk mengalokasikan queue (local/global) ke thread yang memastikan semua thread berjalan secara efisien dengan terus bekerja.

untuk menjalankan goroutine, tambahkan keyword go pada fungsi
> go func_name()

fungsi goroutine tidak bisa mengembalikan nilai, tetapi dapat menggunakan channel

# go routine akan tetap berjalan ketika eksekusi program berakhir

setiap goroutine yang berurusan dengan I/O lambat akan dipending dan mengerjakan goroutine lain.
