# Strconv

Merupakan libary untuk konversi dari dan ke string. Karena type assertion tidak dapat melakukannya. Dan package `strings` hanya bisa tipe lain ke string, bukan sebaliknya.

Memang ada beberapa fungsionalitas yang bisa dilakukan dengan `strings`. Seperti Sprintf-Itoa, Sprintf-FormatFloat. Tetapi kalau untuk kasus single value conversion dan tidak bertujuan untuk menggabungkannya ke string lain. Lebih baik pakai strconv karena performa lebih cepat.

- `AppendQuoteToGraphic(dst []byte, s string) []byte`

- `Atoi(s string) (int, error)` konversi string ke int

- `FormatFloat(f float64, fmt byte, prec, bitSize int) string`

## Strconv vs Strings

### 1. **Kasus cocok pakai `strconv`**

üëâ Saat fokusnya **hanya konversi tipe** (misalnya dari angka ke string untuk penyimpanan atau komunikasi data).

#### Contoh: Menyimpan data ke Redis / JSON

```go
import (
 "fmt"
 "strconv"
)

func main() {
 // Angka yang mau disimpan sebagai string
 userID := 12345

 // Lebih efisien pakai strconv
 key := "user:" + strconv.Itoa(userID)

 fmt.Println("Redis key:", key) // "user:12345"
}
```

üìå Kalau pakai `fmt.Sprintf("user:%d", userID)` juga bisa, tapi `strconv.Itoa` lebih cepat, lebih ringan, dan langsung menunjukkan ‚Äúini konversi tipe‚Äù.

---

### 2. **Kasus cocok pakai `fmt.Sprintf`**

üëâ Saat butuh **format string yang lebih kompleks**.

#### Contoh: Logging / Output terformat

```go
import (
 "fmt"
 "time"
)

func main() {
 user := "andi"
 score := 95.75
 now := time.Now()

 // Lebih natural pakai fmt.Sprintf
 log := fmt.Sprintf("[%s] User %s scored %.2f points", 
  now.Format("2006-01-02 15:04:05"), user, score)

 fmt.Println(log)
 // Contoh output: [2025-08-20 12:34:56] User andi scored 95.75 points
}
```

üìå Kalau pakai `strconv.FormatFloat` + `strings.Join` buat nyusun log, kodenya jadi lebih panjang dan susah dibaca.

---

### Ringkasan

- **`strconv`** ‚Üí cocok untuk konversi sederhana, efisiensi tinggi (misalnya simpan ke DB, encode data).
- **`fmt.Sprintf`** ‚Üí cocok untuk format string yang lebih kompleks (misalnya logging, laporan, UI).
